# Стендиум — плагин Claude Code

Плагин добавляет в Claude Code два скилла для **Стендиума** — витрины-портфолио инструментов:

- **`/stendium-publish`** — собирает карточку инструмента из вашего проекта (что это, под какие задачи, что принимает/отдаёт) и публикует её;
- **`/stendium-story`** — собирает «историю сборки» из git-логов (без секретов/ПДн) и публикует как материал.

Оба показывают текст вам на подтверждение и отправляют на Стендиум по вашему токену. Ничего не публикуется без вашего «да».

Требуется **Node.js 18+** (для глобального `fetch`).

## Установка

### Вариант 1 — локально (быстро, для беты)

Скачайте/скопируйте папку `stendium-plugin` к себе и запустите Claude Code с ней:

```bash
claude --plugin-dir /путь/к/stendium-plugin
```

Либо положите папку под свой каталог скиллов, чтобы она подхватывалась как плагин `stendium@skills-dir` при следующем запуске (папка уже содержит `.claude-plugin/plugin.json`):

```bash
# пример: personal scope
cp -r stendium-plugin ~/.claude/skills/stendium
```

### Вариант 2 — через marketplace (одной командой)

Когда папка плагина опубликована как отдельный публичный git-репозиторий (в ней уже есть `.claude-plugin/marketplace.json`):

```text
/plugin marketplace add <owner>/<repo>
/plugin install stendium@stendium-plugins
```

Проверить, что скиллы видны:

```text
/help              # в списке появятся stendium-publish и stendium-story
/stendium-publish
```

## Настройка токена (один раз)

Публикация идёт под вашим персональным токеном — плагин **не** хранит его.

1. Создайте токен на Стендиуме: **Профиль → Токены** (`https://stendium.ru/profile/tokens`) → «Создать токен». Значение показывается один раз — сохраните.
2. Задайте его в окружении той оболочки, где запускаете Claude Code:

   ```bash
   # bash / zsh
   export STENDIUM_TOKEN="stnd_..."
   ```

   ```powershell
   # PowerShell
   $env:STENDIUM_TOKEN = "stnd_..."
   ```
3. (Опционально) другой адрес API — `STENDIUM_BASE_URL` (по умолчанию `https://stendium.ru`).

Проверить доступ:

```bash
node stendium-plugin/skills/stendium-publish/scripts/publish.mjs --check
```

## Использование

В Claude Code, находясь в папке своего инструмента:

- скажите «**опубликуй мой инструмент на Стендиуме**» — скилл подхватится автоматически, **или**
- вызовите явно: `/stendium-publish`

Дальше скилл:

1. разберётся, что за инструмент (README, код, тип, рабочая ссылка);
2. соберёт карточку и **покажет вам** — вы правите и подтверждаете;
3. опубликует и вернёт ссылку `https://stendium.ru/tool/…`.

Ничего не публикуется без вашего явного «да».

### Обновить уже опубликованное

Если инструмент с этой ссылкой уже опубликован, скилл предложит обновить существующий — понадобится `toolId` (из ссылки на инструмент).

### История сборки (`/stendium-story`)

Отдельный скилл собирает пост «как я это собрал» из git-логов и публикует как материал:

- скажите «**расскажи, как я собрал этот инструмент**» или вызовите `/stendium-story`;
- скилл восстановит историю из коммитов, **вырежет секреты и ПДн** (ключи/токены/пароли/email/телефоны) и покажет очищенный текст;
- после вашего подтверждения — опубликует.

Если в тексте остались секреты, публикация **останавливается** — их нужно убрать. Это защита от случайной утечки ключей из логов.

## Что внутри

```
stendium-plugin/
├── .claude-plugin/
│   ├── plugin.json                     # манифест плагина
│   └── marketplace.json                # для установки через /plugin marketplace
├── skills/
│   ├── stendium-publish/
│   │   ├── SKILL.md                    # публикация инструмента
│   │   └── scripts/publish.mjs         # транспорт: POST/PATCH на /api/tools
│   └── stendium-story/
│       ├── SKILL.md                    # история сборки
│       └── scripts/
│           ├── scrub.mjs               # без секретов/ПДн (гейт)
│           └── publish-story.mjs       # публикация как материал
└── README.md
```

## Приватность

- Токен читается только из переменной окружения `STENDIUM_TOKEN`, в файлы/логи не пишется.
- Публикуется только то, что вы подтвердили. Скилл не отправляет секреты из вашего кода — карточка собирается из описания инструмента, а не из исходников.
